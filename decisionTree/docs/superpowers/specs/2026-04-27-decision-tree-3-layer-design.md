# 决策树三层筛选重构设计 (2026-04-27-decision-tree-3-layer-design)

## 1. 目标
将现有的单层分类决策树页面，严格按照设计图（Frame_143.png）重构为“三层筛选”结构。支持用户逐层、分节点地手动选择筛选条件，展示动态分类过程，并在右侧支持天气预测。

## 2. 核心规则
1.  **数据定义**：共 12 个数据项，对应 4 种天气（雪、雨、晴、阴），每种 3 个。数据属性包括：`isRain` (降水), `isSun` (阳光), `isZero` (零下)。
2.  **筛选条件**：共 3 种可选条件（是否降水、是否有阳光、是否零下）。
3.  **三层树状结构**：
    *   **第 1 层 (Root)**：1 个节点，产生 2 个分支（左是/右否）。
    *   **第 2 层**：2 个节点（左右各 1），每个产生 2 个分支，共 4 个分支。
    *   **第 3 层**：4 个节点，每个产生 2 个分支，共 8 个叶子结果框。
4.  **节点独立与条件互斥**：
    *   **节点独立**：除了根节点（第 1 层），后续层级的每个节点（如第 2 层的左侧节点和右侧节点）都需要**手动单独选择**筛选条件。
    *   **路径互斥**：对于从根节点到任意叶子节点的某一条具体路径，这 3 种条件**不能重复出现**。如果上层节点选择了“是否降水”，其下方所有子节点在选择时，选项中将不再包含“是否降水”。这意味着每条路径上必然包含这 3 种条件的排列组合。

## 3. UI 布局与交互设计
1.  **节点占位与操作**：
    *   未确定条件的节点显示默认样式（如“请选择条件”或默认第一个可用条件），点击可循环切换当前可用的条件。
    *   节点分为：第 1 层（1 个）、第 2 层（2 个）、第 3 层（4 个）。
2.  **分步执行分类**：
    *   提供“开始分类”按钮（或“下一步”按钮）。
    *   点击时，触发**当前所有已就绪且有数据的节点**进行分类动画。
    *   例如：初始时配置好第 1 层节点，点击开始，数据从顶部流向第 2 层的 2 个大框。此时第 2 层的 2 个节点变为可配置状态。配置完成后再次点击，数据流向第 3 层的 4 个中框，以此类推，直到所有数据落入底部的 8 个叶子结果框。
3.  **叶子结果判定**：
    *   由于条件是动态排列组合的，底部的 8 个结果框不能写死标签。
    *   当数据落入最终框时，根据该框所代表的完整路径（例如：降水=是 -> 阳光=否 -> 零下=是）的逻辑组合，动态计算并显示最终天气类型（雪、雨、晴、阴、或根据真值表确定的“无”）。
    *   **判定逻辑 (真值表)**：
        *   降水=是, 阳光=是 -> 无
        *   降水=是, 阳光=否, 零下=是 -> 雪
        *   降水=是, 阳光=否, 零下=否 -> 雨
        *   降水=否, 阳光=是 -> 晴 (忽略零下)
        *   降水=否, 阳光=否 -> 阴 (忽略零下)

## 4. 动画机制
*   **元素分离**：数据图标（`.data-item`）相对于一个绝对定位的容器层进行坐标变换。
*   **两段式轨迹**：
    1.  从当前父节点框中心 -> 移动到连接该父子节点的线段中心点，同时 `scale` 缩小至 0.5。
    2.  从线段中心点 -> 移动到目标子节点框的预定网格位置，同时 `scale` 放大至 1.0。
*   使用 CSS `transition: all 1s ease-in-out` 和 `setTimeout` 编排。

## 5. 右侧预测功能
*   **交互**：用户在右侧面板手动切换三个条件（降水、阳光、零下）的“是/否”。
*   **执行**：点击“重新预测”时。
*   **动画展示**：
    *   一个带有“？”的图标从顶部初始位置出发。
    *   按照左侧决策树**当前已配置好的结构**进行导航。
    *   遇到节点时，读取右侧面板中对应的条件值，决定走向“是”分支还是“否”分支，执行相同的“上->线->下”滚动缩放动画。
    *   如果遇到尚未配置筛选条件的节点，预测过程暂停在当前框内（等待用户在左侧配置并继续），或者预测仅支持在左侧整棵树完全配置完毕后执行（建议强制要求树配置完整后才允许预测完整路径，否则只停留在半路）。*为保证体验，要求左侧树全部配置好后，预测图标才能走到叶子节点。*
    *   到达叶子节点后，将“？”图标替换为根据右侧选择条件计算出的真实天气图标，并更新右侧底部的文本结果。

## 6. 数据结构 (Vue State)
*   **`treeNodes`**: 描述树的结构和状态。
    ```javascript
    {
      id: 'root',
      level: 1,
      selectedFilter: 'rain', // 当前选中的条件
      availableFilters: ['sun', 'zero'], // 给子节点用的剩余条件
      x: 440, y: 100, // 节点中心坐标
      yesChildId: 'node_2_yes',
      noChildId: 'node_2_no'
    }
    ```
*   **`dataItems`**: 12个数据点的数组，增加 `currentNodeId` 追踪其当前所在的节点框。
*   **`predictItem`**: 预测图标的状态（坐标、缩放、可见性、当前节点）。
