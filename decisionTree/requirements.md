## 6. 三层筛选交互规则（核心）

### 6.1 筛选条件集合
可选筛选条件仅 3 种：
- 是否降水
- 是否有阳光
- 是否零下

### 6.2 三层筛选“顺序不固定”
每条路径从根到叶子一共会出现 3 次筛选，但顺序由用户对每个节点的选择决定（不是写死顺序）。

### 6.3 路径互斥规则（必须满足）
对任意一条从上到下的路径：
- 上层节点选择过的筛选条件，下层节点不能再次出现该条件
- 即：同一路径上 3 个节点的筛选条件必须互不重复，最终会形成 `{rain, sun, zero}` 的一种排列组合

### 6.4 节点独立选择（必须满足）
- 第 1 层：根节点独立选择筛选条件
- 第 2 层：左右两个节点 **分别独立选择** 筛选条件（两边可以不同）
- 第 3 层：四个节点 **分别独立选择** 筛选条件（各自受所在路径的互斥规则约束）

### 6.5 “统一点击开始分类”交互（必须满足）
交互目标参考用户给的三张示例图：先在各节点把筛选依据选好，然后点击一次“开始分类”执行分流。

规则：
- 在同一层开始分类前，需要该层所有“可操作节点”都选择好筛选依据
  - 第 1 层：根节点选好后可开始
  - 第 2 层：左右两个节点都选好后可开始
  - 第 3 层：四个节点都选好后可开始
- 点击一次“开始分类”后：本层所有节点同时对各自容器内的数据进行分流动画

## 7. 分类动画（两段式滚动缩放）

每次分流动画要求：
- 数据点从上层容器向下滚动时，沿上下层之间的连接线区域移动
- 动画分两段：
  1) 上层 -> 连接线中点：缓慢缩小（scale 逐步变小）
  2) 连接线中点 -> 下层目标框：缓慢放大（scale 逐步变大）

适用范围：
- 第 1 层分流（顶部 -> 左右大框）
- 第 2 层分流（左右大框 -> 4 个中框）
- 第 3 层分流（4 个中框 -> 8 个叶子框）

## 8. 叶子结果判定（真值表）

结果类型：无/雪/雨/晴/阴

真值表映射关系（已确认正确）：
- 降水=是 且 阳光=是 -> 无
- 降水=是 且 阳光=否 且 零下=是 -> 雪
- 降水=是 且 阳光=否 且 零下=否 -> 雨
- 降水=否 且 阳光=是 -> 晴（零下忽略）
- 降水=否 且 阳光=否 -> 阴（零下忽略）

注意：
- 由于三层筛选顺序不固定，叶子框的结果不能“写死”，必须根据叶子路径实际对应的三要素取值推断结果。

## 9. 右侧预测功能

### 9.1 预测输入
用户在右侧面板选择：
- 是否降水：是/否
- 是否有阳光：是/否
- 是否零下：是/否

### 9.2 预测动画与联动
- 点击“重新预测”后，出现一个“？”图标（`icon_question.png`）
- “？”图标从顶部出发，按照左侧决策树当前节点的筛选依据与右侧选择的值，逐层决定走“是/否”分支
- 预测过程同样使用“上->线->下”的两段式滚动缩放动画
- 到达最终叶子位置后，“？”图标替换为推断出的真实天气图标，并更新右侧的文字预测结果

### 9.3 预测前置条件
- 必须在左侧决策树完成三层配置并完成分类后才允许预测（保证路径完整）
